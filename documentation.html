ORVBA System Documentation


Overview


ORVBA (Online Roadside Vehicle Breakdown Assistance) is a specialized direct-to-mechanic marketplace designed for real-time roadside assistance. It connects vehicle owners (both registered and guests) with verified mechanics, streamlining the process from the initial breakdown report to the final service feedback.



🛠 Technology Stack


  • **Backend Framework**: [Laravel 11](https://laravel.com/) (PHP)
  • **Frontend Library**: [React 18](https://react.dev/)
  • **Frontend Bridge**: [Inertia.js](https://inertiajs.com/) (The "Classic Monolith" feel with a modern SPA experience)
  • **Styling**: [Tailwind CSS](https://tailwindcss.com/)
  • **Real-time / WebSockets**: [Laravel Reverb](https://reverb.laravel.com/) & [Laravel Echo](https://laravel.com/docs/master/broadcasting)
  • **Database**: MySQL
  • **Iconography**: [Lucide React](https://lucide.dev/)



🚀 Core Functionalities


1. Direct Booking Marketplace


  • **Guest Checkout**: Users can book assistance without an account. The system automatically links these bookings to users if they later register with the same license plate.
  • **Direct Assignment**: Users can choose a specific mechanic from the public directory or a general request is handled by the system logic.


2. Service Lifecycle Management


  • **Booking (Pending)**: User submits a request.
  • **Assigned (Accepted)**: Mechanic accepts the request; a `Service` record is created.
  • **In Progress**: Mechanic starts the work.
  • **Completed**: Mechanic finalizes the work, notifying the User and Admin.


3. Real-time Synchronization


  • **Presence Indicators**: Mechanics have real-time "Online/Offline" status lights on the directory, powered by presence channels.
  • **Live Updates**: Dashboards for Users, Mechanics, and Admins refresh automatically when a booking status changes (via WebSocket broadcasting).


4. Feedback System


  • Users can submit text feedback once a service is "Completed".
  • Feedback is displayed in the Mechanic's history and the Admin's moderation panel.


5. Full Responsiveness


  • The platform uses a mobile-first design with custom slide-over drawers for Admin, Mechanic, and User dashboards.



🏗 Backend Architecture (Laravel)


Models & Database


  • **User**: Standard Laravel user augmented with `license_plate` and `city`.
  • **Mechanic**: Contains professional data (`mechanic_license`, `specialization`), verification status, and real-time online status.
  • **Booking**: The initial request. Stores vehicle data, issue description, and owner contact info.
  • **Service**: Generated upon booking acceptance. Tracks the active work phase (`status`) and technician details.
  • **Feedback**: Linked to a `Service`, storing the user's review message.


Controller Logic


  • **BookingController**: Handles the logic for guest/auth bookings and logic for assigning mechanics.
  • **MechanicController**: Manages the mechanic's workflow (Accepting/Rejecting requests, updating work phases).
  • **AdminController**: Provides high-level oversight (Verifying mechanics, managing users, reviewing logs/feedback).
  • **FeedbackController**: Manages user submissions.



🎨 Frontend Architecture (React + Inertia)


The Inertia.js Bridge


Inertia allows the backend to pass data directly as "Props" to React components.


  • **No API needed**: Controllers return `Inertia::render('PageName', [ 'data' => $data ])`.
  • **Seamless Navigation**: Links use `<Link href={...}>` which prevents full page reloads, maintaining the SPA feel.


Component Structure


  • **Layouts**:

- `AdminLayout`: Sidebar-centric with mobile drawer.

- `MechanicLayout`: Workflow-centric with mobile drawer.

- `UserLayout`: History-centric with mobile drawer.

  • **Pages**:

- Located in `resources/js/Pages/`, mapped 1:1 with controller routes.

  • **Shared Components**:

- `Pagination`, `SidebarContent`, `ApplicationLogo`.



📡 Real-time Flow (Broadcasting)


1. **Trigger**: A mechanic updates service status in `MechanicController`.

2. **Event**: A Laravel Event (e.g., `ServiceUpdated`) is dispatched.

3. **Broadcast**: Reverb broadcasts the event over a channel (e.g., `services`).

4. **Listener**: Frontend `useEffect` hooks using `window.Echo` catch the event and trigger `router.reload()` for zero-refresh data updates.



🔧 Installation & Setup


1. `composer install` & `npm install`

2. `php artisan migrate`

3. `php artisan reverb:start` (In a separate terminal)

4. `npm run dev`
